# Plan: Add Sitemap to KJT Website

## Project Analysis
- **Framework**: Vite + React SPA (not Next.js)
- **Build output**: `dist/`
- **Deployment**: Static files hosting
- **Current routes**: Single page app with `/` as the only route
- **Articles**: Stored in markdown files under `public/bfnn_md/`, `public/bfnn_md2/`, `public/bfnn_md3/`, loaded dynamically via modal

## Challenge
This is a SPA where articles open in a modal (not on separate URLs). Without URL routing, a sitemap can only list the homepage.

## Proposed Solution

### Option A: Simple Static Sitemap (Recommended)
Generate `sitemap.xml` with just the homepage. Quick and minimal.

### Option B: Full Dynamic Sitemap
1. Add URL query param support for articles (e.g., `/?id=1234`)
2. Generate `sitemap.xml` at build time with all article URLs

**Decision needed**: Which approach do you prefer?

- **Option A**: Just homepage in sitemap (simple, ~5 min work)
- **Option B**: Full sitemap with all article URLs (requires URL routing + build plugin, ~30 min work)

1. Modify `App.tsx` to read `?id=` query param and open article modal on page load
2. Add Vite plugin to generate `sitemap.xml` at build time with all article URLs
